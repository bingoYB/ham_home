# Cursor Rules - Token Saving

## 通用输出规则
- 不要解释显而易见的代码或编程语言基础
- 不要重复未发生变更的代码
- 除非明确要求，否则不要总结文件内容
- 除非另有要求，只输出变更的代码或 diff
- 避免冗长的推理过程，直接给出结论
- 使用简洁、专业的技术语言
  
## 多任务合并规则
- 能一次完成的任务，禁止拆成多轮对话
- 分类 / 标签 / 描述等必须一次性生成
- 不要在后续轮次重复回顾或复述已有上下文

## 历史与上下文管理
- 不要重复已有对话内容
- 只参考最近、最相关的信息
- 避免回顾整个对话历史
- 不需要“根据以上内容总结”类输出，除非明确要求
