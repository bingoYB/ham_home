# 子任务 03：分类和书签置顶

## 目标

让用户把高频使用的分类和书签固定在侧边栏顶部，减少搜索和层级展开成本。

## 范围

- 分类支持置顶和取消置顶。
- 书签支持置顶和取消置顶。
- 侧边栏新增“置顶”区。
- 置顶内容支持最近置顶优先和手动排序。

## 不包含

- 工作空间置顶。
- 跨设备置顶冲突合并策略，除非当前同步实现要求。

## 用户故事

- 作为用户，我可以把常用分类固定在侧边栏顶部。
- 作为用户，我可以把常用书签固定在侧边栏顶部。
- 作为用户，我可以调整置顶内容顺序。

## 入口

分类入口：

- 侧边栏分类更多菜单。
- 分类管理页更多菜单。

书签入口：

- 侧边栏书签更多菜单。
- 书签管理页更多菜单。
- 书签详情页。

菜单文案：

- `置顶`
- `取消置顶`

## 侧边栏展示

侧边栏顶部新增“置顶”区：

- 置顶分类展示分类名称、图标、数量。
- 置顶书签展示网站图标、标题、域名。
- 无置顶内容时不展示置顶区。
- 置顶内容较多时默认折叠，用户可展开。

点击行为：

- 点击置顶分类：侧边栏定位到该分类，并展示该分类下的书签。
- 点击置顶书签：直接打开网页。

排序：

- 最近置顶的内容排在前面。
- 支持手动排序置顶内容。

## 数据与状态

建议记录：

- 置顶对象类型：分类或书签。
- 置顶对象 ID。
- 置顶时间。
- 手动排序值。

后续扩展：

- 工作空间置顶可复用同一置顶模型。

## 验收标准

- 分类可以置顶和取消置顶。
- 书签可以置顶和取消置顶。
- 侧边栏能展示置顶分类和置顶书签。
- 无置顶内容时不展示置顶区。
- 点击置顶分类能定位并展示分类内容。
- 点击置顶书签能打开网页。
- 最近置顶内容默认排在前面。
- 用户手动排序后能保持排序结果。

## 依赖

- 现有分类管理能力。
- 现有书签管理能力。
- 侧边栏展示和打开书签能力。
