# 子任务 06 人工测试用例：工作空间智能分析

## 前置条件

- 已执行 `pnpm --filter hamhome build` 并加载最新扩展。
- 当前窗口打开 5 个以上页面，建议包含技术文档、代码仓库、搜索结果、后台页面和 1 个重复 URL。
- 设置页中 `智能分类` 可分别测试开启和关闭状态。

## 用例 1：保存时重复 URL 提示与默认去重

1. 打开 `app.html#workspaces`。
2. 点击 `保存当前窗口`。
3. 确认保存弹窗展示重复 URL 提示。
4. 不勾选 `仍然全部保存`，直接保存。

预期结果：

- 工作空间保存成功。
- 工作空间详情中页面数量为去重后的数量。
- 智能分析摘要中 `总计` 与 `去重后` 数量合理。

## 用例 2：保留全部重复页面

1. 再次点击 `保存当前窗口`。
2. 在保存弹窗中勾选 `仍然全部保存`。
3. 保存后打开该工作空间详情。

预期结果：

- 重复 URL 页面全部保留。
- 点击 `开始分析` 后，重复页被标记为不建议转书签。
- 分析摘要展示去重后页面数和分类分布。

## 用例 3：触发智能分析

1. 选择一个未分析的工作空间。
2. 点击详情区 `开始分析`。

预期结果：

- 分析完成后显示推荐名称、推荐标签、总页数、去重后页数、建议转书签数量。
- 页面筛选器中的 `全部 AI 分类` 可选项包含分析出的分类。
- 页面行展示的 AI 分类可用于后续筛选和转书签。

## 用例 4：AI 分类开启与关闭

1. 在设置页开启 `智能分类`，回到工作空间详情点击 `重新分析`。
2. 检查技术文档、代码仓库、搜索结果等页面是否进入对应分类。
3. 在设置页关闭 `智能分类`，再次点击 `重新分析`。

预期结果：

- 开启时，符合规则的页面进入推荐分类。
- 关闭时，未匹配页面进入 `未分类`。
- 分析摘要中的分类分布同步更新。

## 用例 5：分析结果辅助转书签

1. 分析工作空间。
2. 使用 `全部 AI 分类` 筛选到 `技术文档` 或 `代码仓库`。
3. 点击 `转为书签` 并确认。

预期结果：

- 筛选结果只包含对应分类页面。
- 转书签确认框只展示选中的分类页面。
- 搜索结果、登录页、重复页默认不在推荐转书签页面中。
