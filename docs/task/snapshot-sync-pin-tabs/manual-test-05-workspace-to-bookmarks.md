# 子任务 05 人工测试用例：工作空间转书签

## 前置条件

- 已执行 `pnpm --filter hamhome build` 并加载最新扩展。
- 已通过工作空间页面保存至少 1 个工作空间，页面数不少于 4 个。
- 工作空间中至少包含 1 个已经存在于 HamHome 书签库的 URL。
- 已准备至少 1 个分类和 1 个常用标签。

## 用例 1：全部转书签

1. 打开 `app.html#workspaces`。
2. 选择一个工作空间。
3. 点击详情区 `全部转书签`。
4. 在确认弹窗中检查页面列表和已存在书签标记。
5. 选择分类，输入标签。
6. 点击 `确认转为书签`。

预期结果：

- 弹窗关闭并提示新增、跳过、失败数量。
- 已存在的 URL 不会重复创建书签。
- 新增页面出现在书签列表中，分类和标签正确。
- 工作空间页面状态显示 `已转书签` 或 `已存在`。

## 用例 2：选择部分页面转书签

1. 在工作空间详情中取消勾选部分页面，只保留 1-2 个页面。
2. 点击 `转为书签`。
3. 在确认弹窗中选择不同分类和标签后确认。

预期结果：

- 只保存被选中的页面。
- 未选中的页面不改变转书签状态。
- 重复 URL 仍被识别并跳过。

## 用例 3：筛选后转书签

1. 使用域名筛选器选择某个域名。
2. 使用收藏状态筛选器选择 `未收藏`。
3. 勾选筛选后的页面并点击 `转为书签`。

预期结果：

- 页面列表只展示符合筛选条件的页面。
- 转换范围只包含当前选中的筛选结果页面。
- 转换完成后，收藏状态筛选结果会刷新。

## 用例 4：AI 命令推荐

1. 在 AI 命令输入框输入：`把技术文档转成书签，排除搜索结果`。
2. 点击 `AI 推荐`。
3. 检查详情区被勾选的页面。
4. 点击 `转为书签` 并确认。

预期结果：

- AI 推荐只会更新页面勾选状态，不会立即创建书签。
- 用户确认前，书签列表不新增记录。
- 用户确认后，推荐页面才被保存为书签。

## 用例 5：转换失败与重试

1. 手动构造或保留一个不可保存 URL 的工作空间页面。
2. 尝试转书签。
3. 修正或取消该页面后再次转换。

预期结果：

- 失败页面显示失败状态，不影响其他页面成功保存。
- 失败数量在结果提示中体现。
- 用户可以继续转换其他页面。
