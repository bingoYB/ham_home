# 子任务 02：Markdown 快照同步到 Obsidian

## 目标

让用户把 HamHome 已保存的 Markdown 快照保存到 Obsidian，用于后续编辑、整理和备份。

## 范围

- 设置页新增“快照同步”区域。
- 支持配置 Obsidian 保存位置。
- 支持保存书签后自动同步 Markdown 快照。
- 支持单条同步和批量同步。
- 展示同步状态、同步结果和失败原因。

## 不包含

- Obsidian 到 HamHome 的双向同步。
- 非 Markdown 快照同步。
- Obsidian 插件开发。

## 用户故事

- 作为用户，我可以配置 Markdown 快照保存到 Obsidian 的目标位置。
- 作为用户，我可以在保存书签后自动同步快照到 Obsidian。
- 作为用户，我可以对某条书签手动同步或重试同步。
- 作为用户，我可以批量同步已有 Markdown 快照。

## 入口

设置页：

- `启用 Obsidian 保存`
- 保存位置。
- 是否保存书签时自动同步 Markdown 快照。
- 最近同步结果。

单条同步：

- 保存面板：`保存后同步到 Obsidian`
- 书签详情：`同步到 Obsidian`
- 书签管理页单条菜单：`同步到 Obsidian`

批量同步：

- 同步选中书签。
- 同步某个分类下的书签。
- 同步某个标签下的书签。
- 只同步有 Markdown 快照的书签。
- 跳过已同步且未变化的书签。

## 状态与反馈

同步状态：

- `未同步`
- `等待同步`
- `同步中`
- `已同步到 Obsidian`
- `同步失败`

失败处理：

- 显示失败原因。
- 提供 `重试`。
- 不影响书签本身保存。

批量同步反馈：

- 总数。
- 成功数量。
- 失败数量。
- 跳过数量。
- 失败项列表。

## 数据与状态

建议记录：

- Obsidian 保存是否启用。
- 保存目标路径。
- 是否自动同步。
- 单条书签最近同步状态。
- 最近同步时间。
- 最近同步内容版本或快照更新时间。
- 失败原因。

## 验收标准

- 用户能启用或关闭 Obsidian 保存。
- 用户能配置 Obsidian 保存位置。
- 单条 Markdown 快照能同步成功。
- 单条同步失败能看到失败原因并重试。
- 批量同步能展示成功、失败、跳过数量。
- 已同步且未变化的 Markdown 快照能被跳过。
- 非 Markdown 快照不会被误同步。

## 依赖

- 子任务 01 的快照保存和快照状态。
- Markdown 快照生成结果。
- 设置页持久化能力。
