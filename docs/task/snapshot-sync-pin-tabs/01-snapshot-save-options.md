# 子任务 01：快照保存可选

## 目标

让用户在保存书签时控制是否保存页面快照，并能在设置页配置默认快照策略。

## 范围

- 保存面板增加 `保存快照` 开关。
- 设置页增加默认快照策略。
- 书签管理页和详情页提供快照查看、保存、更新、删除操作。
- 保存过程展示书签和快照的独立状态。

## 不包含

- Markdown 快照同步到 Obsidian。
- 工作空间内页面快照。
- 快照存储格式重构，除非现有实现无法支持默认策略。

## 用户故事

- 作为用户，我可以只保存书签信息，不保存离线快照。
- 作为用户，我可以设置默认是否保存快照，减少每次保存时重复选择。
- 作为用户，我可以对已保存书签补充、更新或删除快照。

## 交互要求

保存面板新增“页面快照”区域：

- 开关：`保存快照`
- 开启说明：`保存离线内容，便于后续回看`
- 关闭说明：`仅保存书签信息，减少空间占用`
- 默认状态跟随设置页。
- 本次开关修改不改变设置页默认值。

保存状态：

- `正在保存书签`
- `正在保存快照`
- `书签已保存`
- `书签已保存，未保存快照`
- `快照已保存`
- `快照保存失败，可稍后重试`

设置页新增：

- `默认保存快照`
- `默认快照类型`
  - `自动`
  - `Markdown 优先`
  - `完整 HTML`
  - `默认不保存`

书签管理页和详情页操作：

- `查看快照`
- `保存/更新快照`
- `删除快照`

## 数据与状态

建议最小状态：

- 书签是否有快照。
- 快照保存状态。
- 快照类型。
- 快照最后更新时间。
- 设置页默认快照策略。

状态枚举：

- `未保存快照`
- `快照保存中`
- `已保存快照`
- `快照保存失败`

## 验收标准

- 保存书签时可以关闭快照，关闭后只保存书签信息。
- 设置页默认值能影响保存面板初始状态。
- 保存面板临时修改不会反写设置页默认值。
- 快照保存失败时不影响书签保存成功。
- 已保存书签可以查看、保存/更新、删除快照。
- 用户能区分“书签保存失败”和“快照保存失败”。

## 依赖

- 现有书签保存流程。
- 现有页面快照生成和存储能力。
- 设置页持久化能力。
