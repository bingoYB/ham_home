# 人工测试用例：02 Markdown 快照同步到 Obsidian

## 前置条件

- 已安装开发版 HamHome 扩展。
- 浏览器允许扩展使用下载权限。
- 本机已有 Obsidian Vault，或准备一个可导入到 Obsidian 的下载目录。
- 至少存在一个 Markdown 快照书签。建议在保存面板中将快照类型设为 `Markdown 优先` 后保存网页。

## 用例 1：配置 Obsidian 保存

步骤：

1. 打开设置页，进入 `通用设置`。
2. 在 `快照同步` 区域开启 Obsidian 保存。
3. 将 `Obsidian 保存位置` 设置为 `Obsidian/HamHome`。
4. 开启 `保存书签后自动同步 Markdown 快照`。

预期：

- 开关状态可保存。
- 刷新设置页后，保存位置和自动同步开关保持不变。
- 浏览器下载目录下后续会使用 `Obsidian/HamHome` 作为保存子目录。

## 用例 2：保存面板自动同步 Markdown 快照

步骤：

1. 设置页开启 Obsidian 保存和自动同步。
2. 打开普通可读网页。
3. 打开保存面板，确认 `保存快照` 开启，快照类型为 `Markdown 优先`。
4. 确认 `保存后同步到 Obsidian` 开启。
5. 保存书签。

预期：

- 书签保存成功。
- Markdown 快照保存成功。
- 浏览器下载目录下生成 `Obsidian/HamHome/<书签标题>.md`。
- 生成的 Markdown 文件包含 frontmatter、原始 URL 和正文内容。

## 用例 3：保存 HTML 快照时跳过 Obsidian 同步

步骤：

1. 设置页开启 Obsidian 保存。
2. 打开保存面板，将快照类型设置为 `完整 HTML`。
3. 勾选 `保存后同步到 Obsidian`。
4. 保存书签。

预期：

- 书签和 HTML 快照保存成功。
- Obsidian 同步失败或跳过，并提示仅支持 Markdown 快照。
- 书签本身不回滚、不删除。

## 用例 4：书签管理页单条同步

步骤：

1. 进入书签管理页。
2. 找到一个有 Markdown 快照的书签。
3. 打开更多菜单，点击 `同步到 Obsidian`。
4. 等待提示。

预期：

- 成功时展示 `已同步到 Obsidian`。
- 下载目录生成或覆盖对应 `.md` 文件。
- 再次同步且内容未变化时，可提示已跳过。

## 用例 5：书签管理页失败重试

步骤：

1. 关闭设置页中的 Obsidian 保存。
2. 在书签管理页点击某条书签的 `同步到 Obsidian`。
3. 再回设置页开启 Obsidian 保存。
4. 对同一书签再次点击 `同步到 Obsidian`。

预期：

- 第一次同步失败，并提示 Obsidian 保存未启用。
- 第二次同步成功或按快照类型给出明确失败原因。
- 失败不会影响书签和快照本身。

## 用例 6：批量同步选中书签

步骤：

1. 进入书签管理页。
2. 勾选多个书签，其中至少包含：
   - 一个有 Markdown 快照的书签。
   - 一个无快照的书签。
   - 一个 HTML 快照书签。
3. 点击批量工具栏中的 `同步选中快照`。

预期：

- 批量结果展示成功、失败、跳过数量。
- 有 Markdown 快照的书签同步成功。
- 无快照或非 Markdown 快照的书签展示失败原因。
- 已同步且未变化的书签可被跳过。
