# 子任务 04 人工测试用例：工作空间保存与恢复 MVP

## 前置条件

- 已执行 `pnpm --filter hamhome build` 并加载最新构建的扩展。
- 浏览器中打开至少 3 个普通网页，建议包含 1 个重复 URL，并至少有 1 个已固定标签页。
- 扩展设置中 `enableOmniboxSearch` 保持开启。

## 用例 1：工作空间页面保存当前窗口

1. 打开扩展管理页 `app.html#workspaces`。
2. 点击 `保存当前窗口`。
3. 检查弹窗中页面列表数量、标题、URL、重复 URL 提示是否符合当前窗口。
4. 修改名称、描述，选择分类并输入 1-2 个标签。
5. 点击保存。

预期结果：

- 弹窗关闭并提示 `工作空间已保存`。
- 工作空间列表出现新记录，显示页面数量、分类、标签和主要域名。
- 点击该记录后，详情区展示完整页面列表，所有页面默认被选中。

## 用例 2：搜索、筛选和排序

1. 在搜索框输入工作空间名称中的关键词。
2. 输入某个页面域名关键词。
3. 切换分类筛选到刚保存时选择的分类。
4. 切换排序为 `按最近恢复`。

预期结果：

- 名称、页面域名、标签均可命中工作空间。
- 分类筛选只展示对应分类的工作空间。
- 排序切换不报错，列表仍可点击查看详情。

## 用例 3：恢复工作空间

1. 在工作空间卡片点击 `新窗口恢复`。
2. 返回工作空间页面，取消勾选部分页面。
3. 点击详情区 `当前窗口恢复选中`。

预期结果：

- 第一步会打开新窗口并恢复该工作空间全部页面。
- 第二步只恢复选中的页面。
- 当前窗口恢复时，已存在的相同 URL 会被跳过。
- 恢复完成后工作空间状态变为 `已恢复`，详情区显示最近恢复时间。

## 用例 4：右键菜单和快捷键入口

1. 在普通网页右键，点击 `保存当前窗口为工作空间`。
2. 打开 `app.html#workspaces` 检查新工作空间是否出现。
3. 使用浏览器扩展快捷键设置确认 `保存当前窗口为工作空间` 命令存在。
4. 触发该快捷键后再次检查列表。

预期结果：

- 右键菜单能保存当前窗口。
- 快捷键命令已注册，触发后能新增工作空间。
- 保存失败时不会影响已有工作空间数据。

## 用例 5：地址栏搜索恢复

1. 在地址栏输入 `ham` 后按空格或 Tab 进入 HamHome omnibox 模式。
2. 输入刚保存的工作空间名称或页面域名。
3. 在建议中选择 `Workspace` 结果并回车。

预期结果：

- 地址栏建议包含对应工作空间，展示名称、页面数量和主要域名。
- 回车后在新窗口恢复该工作空间。
- 恢复后再次打开工作空间页面，可看到最近恢复时间更新。

## 用例 6：删除工作空间

1. 在工作空间卡片点击删除按钮。
2. 确认删除。

预期结果：

- 显示删除成功提示。
- 该工作空间从列表中消失，刷新页面后仍不存在。
