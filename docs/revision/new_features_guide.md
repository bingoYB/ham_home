## 批量操作

### 开启批量模式

1. 打开插件 Popup
2. 切换到"列表"标签页
3. 点击右上角"批量"按钮

### 选择书签

- 点击任意书签卡片进行选择/取消选择
- 点击"全选"按钮选中所有书签
- 已选书签会显示蓝色边框和复选框标记

### 批量操作

选中书签后，工具栏显示以下操作：

- **添加标签**：为选中的书签批量添加标签（逗号分隔）
- **更改分类**：将选中的书签移动到指定分类
- **删除**：批量删除选中的书签（软删除）

### 退出批量模式

点击"取消"按钮退出批量模式

## 智能标签推荐

### 前置条件

在设置中配置 AI 并启用"智能标签推荐"功能：
- **设置 → AI 配置 → 智能功能 → 智能标签推荐** ✓

### 使用流程

1. 收藏网页时，保存面板会自动显示"智能推荐"区域
2. 推荐标签显示在蓝色卡片中，包含：
   - 标签名称
   - 推荐理由（hover 查看）
3. 点击任意推荐标签，自动添加到"标签"输入框
4. 可手动点击"获取推荐"按钮重新获取

### 推荐来源

- **AI 推荐**（需配置 API）：基于内容分析
- **规则推荐**（备用）：基于 URL 和关键词

### 示例

访问 `https://github.com/xxx/yyy` 时：
- 推荐标签：开发、代码、GitHub
- 如果内容包含 "React"，会推荐：React

## 智能分类推荐

### 前置条件

在设置中启用"智能分类"功能（默认开启）：
- **设置 → AI 配置 → 智能功能 → 智能分类** ✓

### 使用流程

1. 收藏网页时，保存面板会显示"推荐分类"
2. 推荐分类显示在智能推荐区域，包含：
   - 💡 图标
   - 分类名称
   - 置信度（百分比）
3. 点击推荐分类，自动选中对应分类
4. 如果推荐的是预设分类且不存在，会自动创建

### 推荐逻辑

1. **优先匹配用户已有分类**
2. **其次推荐预设分类**
3. **最后基于关键词匹配**

### 示例

访问技术文章时：
- 如果你有"技术"分类 → 推荐"技术"（80%）
- 如果没有 → 推荐"技术开发"（预设分类）

## 翻译功能

### 开启翻译

在设置中开启翻译功能：
- **设置 → AI 配置 → 智能功能 → 自动翻译** ✓

### 功能说明

开启后，AI 分析生成的内容会自动翻译为中文：
- **标签翻译**：如 "tutorial" → "教程"
- **摘要翻译**：英文摘要 → 中文摘要

### 注意事项

- 翻译需要额外的 API 调用，会增加 Token 消耗
- 如果 AI 本身就返回中文，则不会重复翻译
- 可以随时在设置中开启/关闭

### 适用场景

- 浏览英文网站时
- AI 服务商默认返回英文结果时
- 需要统一使用中文标签时

3. 导入后可在收藏时使用这些分类

- 自动匹配预设分类

## 配置建议

### 推荐配置（高效模式）

```
✓ 启用 AI 分析
✓ 智能标签推荐
✓ 智能分类
✗ 自动翻译（按需开启）
```

**理由**：
- 智能推荐显著提升效率
- 翻译功能会增加 API 调用成本
- 如果 AI 本身返回中文，无需开启翻译

### 推荐配置（国际化模式）

```
✓ 启用 AI 分析
✓ 智能标签推荐
✓ 智能分类
✓ 自动翻译
```

**适用场景**：
- 经常浏览英文网站
- AI 服务商默认返回英文
- 需要统一使用中文

## 常见问题

### Q: 智能推荐不显示？
**A**: 检查以下项：
1. AI 配置是否正确（API Key）
2. 智能功能是否开启（设置 → AI 配置）
3. 网络连接是否正常

### Q: 批量操作后能撤销吗？
**A**: 当前版本暂不支持撤销，但删除的书签会进入回收站（软删除），可以恢复。

### Q: 预设分类可以自定义吗？
**A**: 导入后的预设分类可以重命名、删除，但无法修改预设分类列表本身。

### Q: 翻译功能会增加多少成本？
**A**: 每个书签的翻译约消耗 100-200 tokens，建议按需开启。

### Q: 智能分类准确度如何？
**A**: 
- AI 模式：准确度 80-90%
- 规则模式：准确度 60-70%
- 建议导入预设分类后使用，可提升准确度

## 技术说明

### 数据存储

所有数据存储在本地：
- 书签数据：`chrome.storage.local`
- 快照数据：`IndexedDB`
- 配置数据：`chrome.storage.local`

### 隐私保护

- API Key 仅存储在本地
- 不会上传至任何服务器
- AI 调用直接与服务商通信

### 性能优化

- 批量操作：一次性更新，减少 I/O
- 智能推荐：异步加载，不阻塞保存
- 规则推荐：作为备用方案，无需 API
