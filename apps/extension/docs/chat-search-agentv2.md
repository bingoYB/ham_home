意图识别
- 查询
    - 按时间查询
    - 按分类查询
    - 按标签查询
    - 语义化查询
    - 复合查询（可能包含上面几个或所有）
- 统计：比如昨天收藏了哪些网站
- Help ：询问插件功能用途的，比如快捷键啊，设置啊

如果不在上述意图范围内，比如说用户只输入了某个词句，默认进行语义化搜索

查询要求：
	当用户明确要求说有筛选条件的，比如：xxx 分类下的 xxx， 带有 xxx 标签的 xxx， 昨天收藏的 xxx，才进行这些筛选操作，否则都按语义化查询来
	语义化查询的关键词需要提炼下，不能把用户所有输入都去匹配
		比如： 配眼镜相关的 -> 配眼镜； 我要查询设计素材 -> ue

基于查询的下一步分析

继续精炼搜索（Refine）
场景	推荐
结果 > 20	按时间/域名/标签缩小
结果 < 3	建议用相近关键词
相似度分散	建议用语义搜索/关键词切换

自动整理（Organize）
当结果来自同一主题：
动作	说明
一键打标签	如：#React #State
批量移动	归到某分类
一键复制分享	复制所有链接
